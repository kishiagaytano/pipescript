01
The Problem

The same five steps, on every messy dataset.

Negatives that should be positive, nulls, blanks, wrong types. Before a single insight, you clean — and it's always the same ritual. General-purpose tools can do it, but they wrap a four-word intent in six lines of ceremony.

The ceremony — pandas

import pandas as pd
df = pd.DataFrame(...)
df = df[df.age >= 0]
df = df.dropna()
df = df.fillna(0)
df = df.sort_values(...)

The intent — PipeScript

ages >> removeNegatives()
     >> removeBlanks()
     >> fillNull(0)
     >> sort();

One line. Read left to right — each stage's output feeds the next.

02
What it gives you

A small language with a strong opinion.

>>

The pipeline operator

The lowest-precedence operator in the language, so an entire expression flows cleanly from one stage into the next — the whole design bends around it.

value >> step() >> step() >> done();
Aa

Static typing

Int, Float, Bool, String, Array & Dict — checked before anything runs.

✓

40+ built-in verbs

removeBlanks, fillNull, unique, sort, avg… cleaning with no imports.

{ }

Objects in a pipe

A class method dropped into a pipe auto-maps over every item in a list.

!

Errors that explain

Every error carries its phase, a plain message, and a line. The engine never throws.

03
Under the hood

The interpreter is, itself, a pipeline.

A program flows through the same left-to-right stages the language is built to express. One call — PipeScriptEngine().run() — chains them and returns a single structured result, never an exception.

04
The language

Familiar syntax, one new idea.

Globals and classes sit above the pipeline block. Types, control flow, and objects work the way you'd expect — the only novelty is the pipe.

grader.ps Phase 6–8
// globals and classes live above the pipeline block global Int passingScore = 50; class Grader { String evaluate(Int score) { if (score >= passingScore) { return "PASS"; } return "FAIL"; } } pipeline { local Int[] scores = [72, 40, 88, 15, 63]; Grader g = new Grader(); scores >> g.evaluate(); // a method in a pipe maps over the list }
Result ["PASS", "FAIL", "PASS", "FAIL", "PASS"]

06
Run it yourself

Up and running in four steps.

The compiler is pure Python with no third-party dependencies — you only need FastAPI to run the API locally.

1

Clone the repository

git clone https://github.com/kishiagaytano/pipescript.git
cd pipescript
2

Install the backend dependencies

pip install -r requirements.txt
3

Start the API server

Then open http://localhost:8000/docs to confirm it's live.

uvicorn main:app --reload --port 8000
4

Open the Studio

Double-click pipescript_studio.html — opened from disk it talks to your local backend automatically. Write a program and hit Run.
