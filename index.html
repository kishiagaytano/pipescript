Most data code isn't logic.
It's cleanup.

PipeScript is a minimal, statically-typed language built around one idea — data should flow. Its >> pipeline operator threads a value through each cleaning step, so the code reads exactly like the intent.

Python 3.10+ · pure-stdlib compiler · FastAPI backend · zero-build web IDE

clean_ages.ps 
pipeline {
  local Int[] ages    = [25, -3, null, 41, -9, null];
  local Int[] cleaned = ages >> removeNegatives()
                             >> fillNull(0)
                             >> sort();

  print("Cleaned:", cleaned);
  print("Average:", avg(cleaned));
}
Cleaned: [0, 0, 3, 9, 25, 41]
Average: 13.0

The Problem

The same five steps, on every messy dataset.

Negatives that should be positive, nulls, blanks, values in the wrong type. Before a single insight, you clean — and it's always the same ritual. General-purpose tools can do it, but they wrap a four-word intent in six lines of ceremony.

The ceremony — pandas

import pandas as pd
df = pd.DataFrame(...)
df = df[df.age >= 0]
df = df.dropna()
df = df.fillna(0)
df = df.sort_values(...).reset_index()

The intent — PipeScript

ages >> removeNegatives()
     >> removeBlanks()
     >> fillNull(0)
     >> sort();

One line. Read left to right. Each stage's output feeds the next — no masks, no imports, no reset-index.

What it gives you

A small language with a strong opinion.

>>

The pipeline operator

The lowest-precedence operator, so whole expressions flow cleanly from one stage into the next.

Aa

Static typing

Int, Float, Bool, String, plus Array and Dict — type-checked before a single line runs.

✓

40+ built-in verbs

removeBlanks, removeNegatives, fillNull, unique, sort, avg… data cleaning with no imports.

{ }

Objects in a pipe

Classes, methods, and encapsulation — and a method dropped into a pipe auto-maps over a list.

!

Every error carries its phase, a plain-English message, and a line number. The engine never throws.

Under the Hood

The interpreter is, itself, a pipeline.

A program flows through the same left-to-right stages the language is built to express. One function — PipeScriptEngine().run() — chains them all and returns a single structured result, never an exception.

The Language

Familiar syntax, one new idea.

Globals and classes sit above the pipeline block. Types, control flow, and objects all work the way you'd expect — the only novelty is the pipe.

grader.ps 
// Globals and classes live above the pipeline block.
global Int passingScore = 50;

class Grader {
  String evaluate(Int score) {
    if (score >= passingScore) { return "PASS"; }
    return "FAIL";
  }
}

pipeline {
  local Int[] scores = [72, 40, 88, 15, 63];
  Grader g = new Grader();

  // a method in a pipe runs on every item
  scores >> g.evaluate();   // ["PASS","FAIL","PASS","FAIL","PASS"]
}

Run it Yourself

Up and running in four steps.

The compiler is pure Python with no third-party dependencies — you only need FastAPI to run the API locally.

1

Clone the repository

git clone https://github.com/kishiagaytano/pipescript.git
cd pipescript
2

Install the backend dependencies

pip install -r requirements.txt
3

Start the API server

Then open http://localhost:8000/docs to confirm it's live.

uvicorn main:app --reload --port 8000
4

Open the Studio

Double-click pipescript_studio.html — opened from disk it talks to your local backend automatically. Write a program and hit Run.
